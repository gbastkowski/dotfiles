# Diagnose New `void-function` Warning

## Implementation Steps
1. Capture the exact warning via `emacsclient --eval '(let ((buf (get-buffer "*Warnings*"))) (when buf (with-current-buffer buf (buffer-string))))'` and note the undefined function name plus backtrace entries.
2. Search `emacs/.doom.d/config.el` (and source `emacs/.doom.d/config.org`) for the offending symbol; trace whether it is wrapped in an `after!` block or requires a specific package/module.
3. Inspect `emacs/.doom.d/packages.el` (and any relevant Doom module files) to confirm the package providing that function is installed; plan to add it if missing.
4. Decide on the fix in `config.org`: either wrap the code in `(after! <feature>)`, add `(require '<feature>)`, or replace/remove the call. Keep edits in `config.org` and retangle to regenerate `config.el`.
5. If a package change is needed, update `packages.el` accordingly and schedule `doom sync`.

## Verification
- Re-run `doom sync` if packages changed, otherwise reload via `emacsclient --eval '(doom/reload)'`.
- Recheck `*Warnings*` with the emacsclient form from step 1 to ensure the new `void-function` entry is gone.
- Manually trigger the functionality tied to the fixed code path inside Emacs to confirm it now works without errors.
