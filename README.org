* dotfiles

A [[https://github.com/nix-community/home-manager][Home Manager]]-based dotfiles setup for macOS and Arch Linux.

* Structure

| Directory      | Contents                                      |
|----------------+-----------------------------------------------|
| ~nix/~         | Home Manager flake and configuration          |
| ~bin/~         | Custom scripts deployed to ~~/.local/bin/~    |
| ~emacs/~       | Doom Emacs configuration (~config.org~)       |
| ~claude/~      | Global Claude Code configuration              |
| ~kitty/~       | Kitty terminal configuration                  |
| ~byobu/~       | Byobu configuration                           |
| ~hypr/~        | Hyprland configuration (Arch only)            |
| ~sway/~        | Sway configuration (Arch only)                |
| ~eww/~         | Eww widget system (Arch only)                 |
| ~walker/~      | Walker launcher (Arch only)                   |
| ~wayvnc/~      | Wayvnc configuration (Arch only)              |
| ~systemd/~     | Systemd user services (Arch only)             |
| ~plover/~      | Plover stenography configuration              |
| ~opencode/~    | OpenCode AI configuration                     |
| ~sbt/~         | SBT build tool configuration                  |
| ~zsh/~         | ZSH/p10k configuration                        |
| ~home/~        | Miscellaneous home directory files            |
| ~clang/~       | CMake toolchain files                         |

* Installation

** Prerequisites

- Git (to clone this repo)
- Nix package manager

** macOS

#+BEGIN_SRC sh
  ./bootstrap-darwin.sh
#+END_SRC

This script will:
1. Install Nix (if not already installed)
2. Link ~nix/config/~ to ~~/.config/nix~ (enables flakes)
3. Optionally build and switch to the Home Manager configuration

** Arch Linux

#+BEGIN_SRC sh
  ./bootstrap-arch.sh
#+END_SRC

Same flow as macOS, but installs Nix via ~yay~ and enables the nix daemon.

* Day-to-day Usage

After initial setup, switch to the latest configuration with:

#+BEGIN_SRC sh
  home-manager switch --flake ./nix#gunnar-dotfiles-darwin  # macOS
  home-manager switch --flake ./nix#gunnar-dotfiles-arch    # Arch Linux
#+END_SRC

** Collision handling

Home Manager refuses to overwrite unmanaged files.
Use the ~-b backup~ flag on first switch to let it keep backups of existing files:

#+BEGIN_SRC sh
  home-manager switch -b backup --flake ./nix#gunnar-dotfiles-darwin
#+END_SRC

** Validating without activating

#+BEGIN_SRC sh
  home-manager build --flake ./nix#gunnar-dotfiles-darwin
#+END_SRC

* Home Manager targets

| Target                    | Platform              |
|---------------------------+-----------------------|
| ~gunnar-dotfiles-darwin~  | macOS (~aarch64-darwin~) |
| ~gunnar-dotfiles-arch~    | Arch Linux (~x86_64-linux~) |

* Notes

- ~emacs/.doom.d/~ is linked file-by-file so that ~~/.doom.d~ remains a writable directory (required by ~doom sync~).
- ~emacs/.emacs.doom/~ (Doom Emacs itself) is a git submodule, managed outside Home Manager.
- The ~ista/~ directory is a private git submodule with work-specific configuration.
- ~nix/config/nix.conf~ enables flakes and must be linked to ~~/.config/nix~ before running any nix commands. The bootstrap scripts handle this.
