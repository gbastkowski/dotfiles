* dotfiles
A [[https://github.com/nix-community/home-manager][Home Manager]]-based dotfiles setup for macOS and Arch Linux.

* Installation
** Prerequisites
- Git (to clone this repo)
- Nix package manager

** Bootstrap
Run the single bootstrap script — it detects the OS automatically:

#+BEGIN_SRC sh
  ./bootstrap.sh
#+END_SRC

The script will walk through each step interactively:
1. Install Nix (~curl~ on macOS, ~yay~ on Arch)
2. Enable the Nix daemon (Arch only)
3. Link ~config/~ to ~~/.config/nix~ (enables flakes)
4. Build and switch to the Home Manager configuration
5. Clone and install Doom Emacs into ~~/.emacs.doom/~

** Collision handling
Home Manager refuses to overwrite unmanaged files.
Use the ~-b backup~ flag on first switch to let it keep backups of existing files:

#+BEGIN_SRC sh
  home-manager switch -b backup --flake .#darwin-dotfiles
#+END_SRC

** Validating without activating
#+BEGIN_SRC sh
  home-manager build --flake .#darwin-dotfiles
#+END_SRC

* Day-to-day Usage
Run the upgrade script to pull the latest dotfiles, apply the Home Manager configuration, and update Doom Emacs:

#+BEGIN_SRC sh
  ~/.local/bin/system-upgrade.sh
#+END_SRC

* Nix & Home Manager Primer
A brief glossary for the concepts used in this repo.

** Nix
A purely functional package manager and build system.
Packages are built in isolation and stored in the /nix/store (e.g. ~/nix/store/abc123-emacs-29.4/~).
Nothing is installed globally; everything is linked or referenced from there.

** Flake
A self-contained Nix project with pinned inputs and declared outputs.
~nix/flake.nix~ is this repo's flake — it declares all external dependencies (nixpkgs, home-manager, powerlevel10k, etc.) and the two Home Manager configurations as outputs.
~nix/flake.lock~ pins every input to an exact git revision, making builds reproducible.

** Flake input
An external source (git repo, tarball) that the flake depends on.
Inputs with ~flake = false~ are plain repos without their own ~flake.nix~, consumed as raw file trees (e.g. chemacs2, powerlevel10k).

** Home Manager
A tool that applies a Nix configuration to a user's home directory.
It manages dotfiles, shell config, and some programs declaratively — the ~home.file~ attribute set maps source files to ~$HOME~ paths as read-only symlinks into the Nix store.

** home.file
Declares a symlink from a source file (in the repo or Nix store) into ~$HOME~.
Files managed this way are read-only; Home Manager owns them.

** programs.*
Higher-level Home Manager modules (e.g. ~programs.zsh~, ~programs.tmux~, ~programs.git~) that generate config files and wire up shell integration automatically.

** Activation
Running ~home-manager switch~ compares the current generation with the new one and atomically applies the diff: it creates/removes symlinks, installs packages, and runs any activation scripts.

* Notes
- ~emacs/.doom.d/~ is linked file-by-file so that ~~/.doom.d~ remains a writable directory (required by ~doom sync~).
- ~emacs/.emacs.doom/~ (Doom Emacs itself) is cloned independently by ~bootstrap.sh~, not managed by Home Manager.
- ~ista-dotfiles~ is a private repo included as a Nix flake input (not a submodule).
- ~config/nix.conf~ enables flakes and must be linked to ~~/.config/nix~ before running any nix commands.
  ~bootstrap.sh~ handles this.
- After a gcc or libgccjit upgrade, re-run ~doom env~ to refresh the environment snapshot for GUI Emacs.
